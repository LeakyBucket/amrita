Macros

fact(description)

A fact without a body is a pending fact. Much like a TODO. It prints a reminder when the tests are run.

Example

fact "something thing I need to implement at somepoint"
Source

fact(description, provided // [], var // quote() do _ end, contents)

A fact is the container of your test logic.

Example

fact "about addition" do
  ...
end

If you are using mocks you can define them as part of your fact.

Example

fact "about mock", provided: [Flip.flop(:ok) |> true] do
  Flip.flop(:ok) |> truthy
end

You can optionally examine meta data passed to each fact. Useful when used with callbacks:

Example

setup do
  {:ok, ping: "pong"}
end

fact "with meta data", meta do
  meta[:pong] |> "pong"
end
Source

facts(description, _ // quote() do _ end, contents)

facts are used to group with a name a number of fact tests. You can nest as many facts as you feel you need.

Example

facts "about arithmetic" do
  fact "about addition" do
    ...
  end
end
Source

future_fact(description, _ // quote() do _ end, _)

A future_fact is a pending fact. Its body is NEVER run. Instead it simply prints an reminder that it is yet to be run.

Example:

future_fact "about something that does not work yet" do
  ..
end
